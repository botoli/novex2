# Инструкция по проверке конфигурации Vercel

## 1. Проверка подключения домена
1. Откройте [Vercel Dashboard](https://vercel.com/dashboard)
2. Выберите проект `novextask`
3. Перейдите в раздел **Domains**
4. Убедитесь, что домен `novextask.ru` подключен и имеет статус **Valid**

## 2. Проверка переменных окружения
1. В проекте перейдите в **Settings** → **Environment Variables**
2. Убедитесь, что есть переменная:
   - `VITE_API_URL` = `https://api.novextask.ru`
3. Если переменной нет, добавьте её:
   - Name: `VITE_API_URL`
   - Value: `https://api.novextask.ru`
   - Environment: **Production** (и **Preview** если нужно)

## 3. Настройка DNS записей
1. Перейдите в панель управления вашего DNS-провайдера (где зарегистрирован домен `novextask.ru`)
2. Добавьте A-запись для поддомена `api`:
   - Type: `A`
   - Name: `api`
   - Value: `94.29.6.121` (IP вашего сервера)
   - TTL: `3600` (или автоматически)

## 4. Проверка сборки
1. В Vercel Dashboard перейдите в **Deployments**
2. Убедитесь, что последний деплой успешен
3. Проверьте логи сборки на наличие ошибок

## 5. Проверка работы фронтенда
1. Откройте сайт `https://novextask.ru`
2. Проверьте, что сайт загружается (не бесконечно грузится)
3. Откройте DevTools (F12) → вкладка Network
4. Убедитесь, что запросы идут на `https://api.novextask.ru/api/...`, а не на localhost

## 6. Дополнительные настройки
- **CORS**: Убедитесь, что бэкенд разрешает запросы с `https://novextask.ru`
- **HTTPS**: Vercel автоматически предоставляет SSL-сертификат
- **Редиректы**: При необходимости настройте редиректы в `vercel.json`

## 7. Тестирование API
После настройки DNS проверьте доступность API:
```bash
curl https://api.novextask.ru/api/projects
```

Если ответ 200 OK или 401 Unauthorized (без ошибок сети) — API доступен.

## 8. Если проблемы
1. **Сайт грузится бесконечно**: Проверьте консоль браузера на ошибки CORS
2. **API недоступен**: Проверьте DNS-пропагацию (может занять до 24 часов)
3. **Ошибки сборки**: Проверьте логи Vercel, возможно нужны дополнительные переменные окружения
